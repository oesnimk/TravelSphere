1. Home 화면 (메인 홈화면)
1-1. 화면 목적
사용자가 앱에 진입했을 때
오늘 기준으로 가장 적합한 로컬 문화·역사 코스를 빠르게 발견하도록 돕는 허브 화면.
AI 추천, 카테고리 필터, 주변 명소 요약 정보를 한 번에 제공.
1-2. 상단 영역
상단 인사/타이틀
문구: Where do you want to go?
기능: 별도의 기능은 없고, 서비스 컨셉을 알려주는 타이틀.
검색 바
위치: 타이틀 바로 아래.
입력 방식: 텍스트 검색: 도시명, 장소명, 키워드(예: 궁궐, 미술관, street art 등) 입력. 	       마이크 아이콘: 음성 검색 기능
동작: 검색 결과는 Explore형 리스트 화면 혹은 지도 화면으로 이동해 결과 표시.		  검색어 자동완성(도시, 문화유산, 지역 음식 키워드 등) 제공 가능.
로그인 / 프로필 아이콘
위치: 우측 상단 원형 아바타 영역.
기능: 탭 시 로그인/마이페이지로 이동. 로그인 상태에서는 프로필 이미지, 비로그인 상태에서는 기본 아이콘 노출.
1-3. 중앙 콘텐츠 영역
Explore 섹션
와이어프레임 상 “Explore Now / Discover More” 두 개 카드 탭.
기능: Explore Now: 현재 위치, 시간, 날씨 등을 반영한 즉시 체험 가능한 로컬 문화 코스 카드.   Discover More: 조금 더 긴 체류 시간 기준의 심화 문화 탐방 코스 카드.
각 카드 내 정보:
코스 이름 (예: ‘Nature Trail Exploration’, ‘Cultural Heritage Tour’)
구성 스팟 수 (예: 3 spots, 5 spots)
예상 소요 시간 (예: approx 2 hours)
인터랙션: 카드 탭 시 AI 추천 코스 화면 또는 코스 상세 화면으로 이동.
AI 연동 포인트: 사용자 취향 태그(역사, 현대문화, 건축, 박물관 선호 등)와 현재 위치를 			      기반으로 상위 1~2개 코스를 자동 추천.
Filter by: 카테고리 필터 버튼
버튼 구성 (예시):
History, Culture, Hidden Gems, Temples
필요시 Food, Cafe, Local Market 등 음식/생활 카테고리는 보조 필터로 추가.
기능: 하나 이상 선택 가능.
로컬 문화/역사 중심 반영:
기본 선택값을 History, Culture, Hidden Gems로 두고
음식 관련 카테고리는 세부 옵션으로만 위치시켜 “문화와 역사”가 첫인상에 드러나게 함.
Today’s Thematic Course (오늘의 테마 코스)
기능: 날짜, 요일, 날씨, 로컬 이벤트 등을 반영한 오늘 전용 테마 코스 1개 노출.
테마 예시:
“도심 속 옛길–사찰 투어”
“근대 건축과 독립운동사 코스”
“미술관–갤러리 산책 + 주변 전통 디저트 맛보기” (음식 보조 활용)
인터랙션: 전체 영역 탭 시 코스 상세 페이지.
AI 연동 포인트: 사용자 과거 기록(다녀온 장소, 저장한 장소, 좋아요)을 토대로 테마 선정.
Nearby Attractions (주변 명소 리스트)
구성: 리스트 항목마다 명소 이름 (예: Gyeongju Historic Areas), 한 줄 설명 (예: UNESCO World Heritage Site, Famous Buddhist Temple 등), 현재 위치로부터 거리 또는 예상 이동 시간
기능: 항목 탭 시 장소 상세 화면으로 이동, “Nearby Attractions” 영역 전체를 탭하면 지도 화면으로 이동하며 주변 명소 핀 강조.
1-4. 하단 네비게이션 바
탭 구성: Explore / Guide / Home / AR
기능:
Home: 현재 화면.
Explore: 전체 탐색 리스트 화면으로 이동.
Guide: AI 코스 추천 및 텍스트 가이드 화면으로 이동.
AR: 바로 On-site AR Guide 또는 AR 카메라 진입 선택 화면으로 이동.

2. Explore 화면 (탐색 리스트)
2-1. 화면 목적
전체 도시·장소를 리스트/카드 형식으로 브라우징하며 탐색하게 만드는 화면.
홈보다 검색·필터에 집중한 구조.
2-2. 상단 영역
페이지 타이틀
문구: Where do you want to go? = 단순 안내 문구, 기능 없음.
검색 바
Home 화면과 동일하게 텍스트/음성 검색 지원.
여기서는 검색 결과를 즉시 아래 리스트에 반영.
2-3. Popular Places 섹션
탭형 카테고리
Nature / Cultural / Urban 등 상단 탭.
탭 전환 시 카드 목록의 필터가 바뀜.
Cultural 탭에는 사찰, 궁궐, 박물관, 갤러리 등 문화·역사 중심 스팟이 우선 노출.
카드 리스트
각 카드 구성:
상단: 카테고리(예: Waterfalls, Temples 등)
중간: 장소 이름, 사진 썸네일
하단: 한 줄 설명(예: A blend of history…, Amazing views 등)
기능: 카드 탭 시 장소 상세 화면으로 이동. long-press 혹은 별 아이콘으로 북마크 기능 추가 가능(선택사항).
2-4. Curated for You 섹션
기능: AI가 사용자 취향에 맞게 골라준 장소 리스트.
리스트 항목:
장소 이름
위치(도시/지역명)
카테고리 태그 (예: Historic sites, Local market, Traditional neighborhood 등)
AI 연동:
사용자의 필터 선택, 검색 기록, AR 체험 이력, 저장한 장소 등을 학습하여 노출 순서 결정.
로컬 문화/역사 선호도가 높을수록 해당 카테고리 스팟 비중을 높임.
인터랙션: 항목 탭 시 장소 상세 화면.

3. Map 화면 (지도)
3-1. 화면 목적
사용자의 현재 위치 기준으로 주변 문화·역사 스팟을 지도 위에서 직관적으로 탐색하게 하는 화면.
향후 AR 모드와 긴밀하게 연결될 핵심 기능.
3-2. 상단 검색/필터 영역
검색 바
도시, 장소, 키워드 검색.
검색 시 지도 중심이 해당 결과 위치로 이동.
카테고리 필터 칩
예시: Museums, Galleries, Historic Spots, Eats 등.
기능: 다중 선택 가능, 선택에 따라 지도 위 아이콘 타입/색상 다르게 표시 가능.
설계 방향:
Museums, Galleries, Historic Spots를 기본 활성화.
Eats는 보조 역할로, 문화 동선 내에서 “근처 음식 옵션” 정도로 사용.
3-3. 지도 영역
POINT 아이콘
각 문화/역사 장소, 음식점, 체험 공간에 대한 핀 표시.
핀 유형:
문화유산(사찰, 궁궐 등)
박물관/미술관
현대문화 공간(예술촌, 스트리트 아트 구역)
음식(전통 음식점, 디저트 등 = 서브 아이콘)
기능: 핀 탭 시 하단에 장소 요약 카드가 슬라이드업.
현재 위치 버튼
우측 하단(또는 우측)의 원형 버튼.
탭 시 지도가 현재 GPS 위치로 재정렬.
지도 도구 버튼 (와이어프레임의 세 개 아이콘 영역)
레이어 버튼: 위성/일반/AR경로 등 지도 스타일 전환.
경로 버튼: 선택한 장소까지의 도보/대중교통 경로 표시.
AR 버튼: 현재 선택한 경로/장소 기준으로 AR 네비게이션으로 전환.
3-4. 하단 장소 요약 카드
내용:
장소 이름
간단 설명
문화/역사 태그 (예: Joseon Dynasty, UNESCO 등)
거리/예상 이동 시간
기능:
카드 스와이프로 인근 다른 장소 탐색.
카드 탭 시 장소 상세 화면으로 이동.
3-5. 하단 네비게이션
Explore / Guide / Home / AR 탭 동일.

4. Place Detail 화면 (장소 설명)
4-1. 화면 목적
개별 장소에 대해 스토리 중심의 상세 정보 + 실제 방문에 필요한 정보를 모두 제공.
AI/AR 기능이 실제로 붙는 화면.
4-2. 상단 콘텐츠 영역
대표 이미지 / 미디어
큰 이미지 (슬라이드 가능).
AR 미리보기 섹션을 포함할 수도 있음.
상단 액션 버튼: 뒤로가기(좌상단), 공유, 저장(북마크), 좋아요 등(우상단)
4-3. 기본 정보 영역
장소 이름: 예: Namsan Seoul Tower
오디오 가이드 버튼(선택)
Play Audio Guide
기능: 해당 장소의 역사·문화적 설명을 음성으로 재생. 다국어 지원 (한국어/영어/일본어 등).
텍스트 설명
역사적 배경, 상징성, 주요 이벤트, 건축양식 등
로컬 문화·역사 중심 정보.
스크롤 가능한 본문.
추가 정보 링크: “자세히 보기” 버튼 등으로 세부 글/관련 기사 링크.
4-4. AR / 가이드 연동 영역
AR 모드 진입 버튼
예: “AR로 보기” 또는 “AR Heritage Mode”
기능: 현재 장소에 대한 AR 오버레이 체험 시작. 예: 옛 건물 복원, 스토리 캐릭터 등장, 과거–현재 비교 등.
코스에 추가 버튼
“나의 코스에 추가” 또는 “이 장소를 포함한 코스 보기”
기능: AI 추천 코스에 이 장소를 포함한 대체 경로 제안.

5. Guide Home 화면 (가이드 홈)
5-1. 화면 목적: 텍스트 기반 또는 카드 기반으로 도시 단위의 로컬 문화·역사 가이드와 AI 추천 코스를 제공하는 허브.
5-2. 상단 도시/지역 선택
예: Seoul, South Korea
기능: 도시명을 탭하면 도시 선택 검색 화면으로 이동. 선택 도시 기준으로 가이드·코스·AR 정보 갱신.
5-3. 추천 코스 및 랜드마크 리스트
Recommended Landmarks 섹션
카드 리스트:
예: Namsan Tower, Gyeongbokgung Palace, Cheonggyecheon 등.
각 카드에 짧은 설명과 태그.
Start Course / Plan This Tour 버튼
Start Course: 미리 정의된 코스를 그대로 시작.
Plan This Tour: 선택한 랜드마크들을 기반으로 사용자가 코스를 재구성 가능
5-4. AI 추천 문구/설명 영역
예: 도시 소개 한 단락과 간단한 제안:
“남산을 포함한 서울의 1박2일 코스를 짜줘”
기능: 단순 텍스트지만 이후 챗봇형 인터페이스(질문–답변)로 확장 가능.

6. AI Recommended Course 화면 (가이드_AI 추천 코스)
6-1. 화면 목적
사용자의 취향, 시간, 위치, 일정 조건을 반영한
AI 기반 테마 코스들을 리스트로 보여주는 화면.
6-2. 추천 코스 리스트
각 코스 카드 항목:
코스 이름 (예: “1일 근대건축 코스”, “도심 속 예술–사찰 투어”)
포함 장소 수
예상 소요 시간
이동 방식 아이콘(도보, 대중교통 등)
AI 로직:
입력값: 사용자 선호(역사/현대/자연 등), 현재 위치, 방문 가능 시간대, 날씨.
출력: 상위 3~5개의 템플릿 코스.
6-3. 코스 상세로 진입
카드 탭 시:
코스 상세 화면에서
타임라인 형태로 순서,
각 장소 간 이동 시간,
AR 지원 여부 표시.
“Start this course” 버튼:
시작 시 Map 화면에서 첫 장소로 네비게이션 세팅,
AR 모드로 바로 전환할 수 있는 옵션 제공.

7. On-site AR Guide 화면 (현장 AR 가이드)
7-1. 화면 목적
실제 장소(박물관, 미술관, 유적지)에 도착한 사용자가
스마트폰 카메라만으로 AR 기반 문화 해설을 받을 수 있게 하는 화면.
7-2. AR 카메라 뷰
화면 대부분을 카메라 뷰가 차지.
위에 오버레이로 다음 정보 표시:
현재 인식된 포인트 이름
짧은 설명
“자세히 보기” 또는 “스토리 보기” 버튼
기능:
특정 오브젝트(전시품, 건물 일부)에 카메라를 향하면
AR 마커/마커리스 인식으로 이름과 설명을 띄움.
3D 모델 또는 애니메이션으로 과거 모습, 복원 모델, 인물 등장 등 표현.
7-3. 하단 컨트롤 영역
버튼 예시:
스토리 모드: 장소에 관한 서사형 AR 연출 자동 재생.
텍스트 보기: 화면 하단에 텍스트 설명 펼침.
오디오 해설(선택사항): 음성 가이드 on/off.
7-4. 위치 기반 트리거
GPS와 실내 위치(가능하면 비콘, QR 등) 기반으로
현재 사용자가 어떤 코스의 몇 번째 포인트에 있는지 인식.
코스 진행 상태 표시: “3/7번째 포인트입니다” 같은 진행 바.

8. 부가 기능 설명 (텍스트 스펙에서 언급된 기능 반영)
테마 기반 맞춤 코스 추천
사용자 선호, 위치, 일정, 날씨를 입력값으로 받아 “문화·역사 중심 테마 코스”를 자동 생성.
예:
1일 근대건축 코스
도심 속 예술–사찰 투어
숨은 전시 루트
포인트:
AI 추천의 핵심은 “문화·역사 테마”
음식은 “코스 중간에 들를 수 있는 로컬 음식점/카페 제안” 정도로 보조 배치.
지도 및 위치 기반 문화 탐색
현재 위치 주변의 유적·박물관·전시 정보를 지도 한 화면에서 확인.
GPS 기반 실시간 지도,
거리·시간·이동수단 정보,
코스와 장소 요약 카드 + 바로가기 버튼 제공.
네이버 지도나 Google maps api 사용 예상
다국어 해설 및 실시간 번역
각 장소별 텍스트/오디오 해설 다국어 지원.
안내문 번역은 후순위 기능이지만, 기획 상 명시 가능: 카메라로 안내판을 비추면 사용 언어로 번역
현장 AR 가이드
박물관, 미술관, 유적지 등 특정 공간에서만 활성화되는 AR 모드.
핵심 포인트:
카메라를 통해 유적 실제 모습 위에 연표, 디지털 정보, 복원 그래픽 중첩.
GPS 혹은 실내 위치 기술을 기반으로 가이드 동선 제공.
